A03:2021 – Injection
Esta vulnerabilidad puede permitir que algun atacante pueda enviar datos maliciosos mediante un input 
dentro de la Api. Pudiendo si burlar o saltarse la seguridad de los datos y a su vez recolectando 
dicha informacion de una manera muy facil y sencilla.

Para esta solucion optamos por usar consultas parametrizadas permitiendo usar %s como marcador de posicion para los 
valores y pasando los datos en una tupla separada al método cursor.execute(). Esto garantiza que la base de datos trate el input como datos, no como comandos SQL, neutralizando el ataque de inyeccion.

El cambio se aplico en las funciones Registro, Login, Imagen y Descargar
